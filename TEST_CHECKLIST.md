# JamalC2 最终测试清单

## 环境准备

- [ ] 启动 Server：`cd server && npm run tauri dev`
- [ ] 创建监听器：设置 → 输入端口 → 创建

---

## 1. Implant 生成

- [ ] 在 GUI 中点击 **生成**
- [ ] 选择刚创建的监听器
- [ ] 输入服务器地址（本地测试用 `127.0.0.1`）
- [ ] 点击生成，等待编译完成
- [ ] 确认生成成功

---

## 2. 本地连接测试

```powershell
cd d:\project1\jamalC2\implant\target\release
.\implant.exe -k 321
```

- [ ] Implant 正常上线
- [ ] 服务端显示新客户端
- [ ] 显示正确的用户名、主机名

---

## 3. Shell 命令执行

双击客户端打开 Shell：

| 命令 | 预期结果 |
|------|----------|
| `whoami` | 返回当前用户名 |
| `ipconfig` | 返回网络配置（中文正确显示） |
| `ping 127.0.0.1 -n 2` | 返回 ping 结果 |

- [ ] 命令正常执行
- [ ] 中文输出正确显示

---

## 4. 文件管理

- [ ] 点击 **文件** 按钮
- [ ] 浏览目录正常
- [ ] 下载文件成功
- [ ] 上传文件成功
- [ ] 删除文件成功

---

## 5. 心跳间隔调节

- [ ] 点击 **心跳间隔** 按钮
- [ ] 输入新间隔（如 60）
- [ ] 确认客户端列表显示更新（显示 `60秒 (±20%)`）

---

## 6. 断开连接

- [ ] 点击 **断开连接**
- [ ] 确认提示信息显示
- [ ] 等待 Implant 自动退出

---

## 7. ngrok 测试（可选）

```powershell
ngrok http 4444
```

- [ ] 修改 Implant 配置为 ngrok 地址
- [ ] 重新生成 Implant
- [ ] 确认通过 ngrok 正常上线

---

## 8. 启动密钥测试

```powershell
# 错误密钥
.\implant.exe -k wrong

# 无参数
.\implant.exe
```

- [ ] 两种情况都应该导致 Implant 自删除
- [ ] 确认 exe 文件被删除
